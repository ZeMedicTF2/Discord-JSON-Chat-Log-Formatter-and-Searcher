# Discord-JSON-Chat-Log-Formatter-and-Searcher

Three small Python scripts to work with Discord-exported chat logs:

- **parser.py**: converts `Input/*.json` into readable transcripts in `Chats/*.txt`
- **searcher.py**: searches across `Chats/*.txt` with a CLI and writes `search_results.txt`
- **users.py**: summarises unique users from `Input/*.json` and writes `users.txt`

---

## Quick start

### 1) Put exports in `Input/`

Place one or more Discord-export `.json` files inside the `Input/` folder.

### 2) Generate transcripts

```text
python parser.py

This will:

• read every Input/*.json

• write Chats/<same-name>.txt for each file

• overwrite existing Chats/*.txt if they already exist

• output messages in chronological order (oldest first, newest last)
```

### 3) Search transcripts

```text
python searcher.py

This will:

• scan every Chats/*.txt

• prompt you for search filters in the CLI

• overwrite search_results.txt

search_results.txt contains:

• the filters used

• a per-file summary (counts + line numbers)

• total matches + total characters

• matched messages grouped by file, sorted by timestamp
```

### 4) Generate a users report

```text
python users.py

This will:

• scan every Input/*.json

• overwrite users.txt

users.txt contains:

• a global list of unique users (global name + total entries + username + ID)

• a per-file breakdown (global name + entries in that file)
```

## Search behaviour

### Content contains: substring search (default)

If you input:

```text
target
```

It performs a case-insensitive substring search.


### Content contains: exact token search (quoted)

If you input:

```text
"target"
```

It performs a case-insensitive exact token search.

Matches:

```text
• target

• TARGET

• TaRgEt
```

Rejects:

```text
• targets

• atarget

• ta rget

• TaRGe T
```

## Customisation

### Name replacements

Both parser.py and users.py support replacing specific global names using:

```python

ENABLE_NAME_REPLACEMENTS = True
NAME_REPLACEMENTS = {
    "zemedic": "Medic",
    # "X": "x",
}
```
